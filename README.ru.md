# Glimpse

[English](README.md) | **Русский**

![Plugin screenshot](/img/glimpse.jpg)

Плагины Total Commander для работы с кадрами видео. 
* **WLX** (Lister) отображает равномерно распределённые кадры видео при предпросмотре файла.
* **WCX** (Packer) представляет видео как виртуальный архив изображений-кадров, позволяя выполнять пакетное извлечение стандартными файловыми операциями TC.

Плагины используют возможности [ffmpeg](https://www.ffmpeg.org/) и требуют его наличия и доступности.

## Плагин WLX (Lister)

Обеспечивает мгновенный визуальный обзор содержимого видео без открытия медиаплеера.

### Настройка

Все настройки хранятся в файле `Glimpse.ini` в каталоге плагина. Диалог настроек открывается по F2 или через контекстное меню (правый щелчок). Диалог разделён на вкладки: **General**, **Sampling**, **Appearance**, **Save**, **Clipboard**, **Cache**, **Thumbnails**, **Quick View**, **Hotkeys**. Нажмите **Apply**, чтобы применить изменения к открытому окну просмотра, не закрывая диалог — так живой просмотр работает как предпросмотр. **Действие Apply нельзя отменить кнопкой Cancel.**

#### Sampling

Управляет тем, *какие* моменты видео превращаются в кадры — независимо от параметров движка на вкладке General.

| Параметр                    | По умолчанию | Описание                                                                                                                                                                                                                                                                                    |
|-----------------------------|--------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Skip edges                  | 2%           | Процент длительности видео, пропускаемый в начале и конце, чтобы избежать чёрных заставок в начале/конце                                                                                                                                                                                    |
| Start from random positions | Выкл.        | Когда включено, при открытии файла смещения кадров выбираются случайно внутри их отрезков, а не в детерминированных серединах.                                                                                                                                                              |
| Randomness                  | 50%          | Сила окна случайного разброса на каждом отрезке. 1% слегка сдвигает смещение от центра; 100% позволяет выбрать кадр в любом месте его отрезка.                                                                                                                                              |
| Cache random frames         | Выкл.        | Когда выключено, случайные извлечения читаются из кэша (так что ранее закэшированный случайный выбор по-прежнему срабатывает), но новые выборки обратно не записываются — так каждый Shuffle остаётся по-настоящему новым. Когда включено, случайные выборки кэшируются так же, как обычные |

Кнопка **Refresh** на панели инструментов — это раздельная кнопка: щелчок по ней повторно извлекает текущие смещения, а стрелка выпадающего списка открывает пункт **Shuffle** (Ctrl+R), который заново выбирает смещения и повторно извлекает кадры. Shuffle работает независимо от флажка *Start from random positions* — этот переключатель управляет только поведением по умолчанию при открытии файла.

##### Шаблон строки состояния

Строка состояния строится из строки токенов, в которой каждый `%name%` становится одной панелью в порядке записи. Пробелы между токенами игнорируются. Неизвестные идентификаторы отображаются как их исходный текст, чтобы опечатки были видны.

Шаблон по умолчанию (воспроизводит расположение панелей, использовавшееся до появления этой функции):

```
%file_position%%frame_position%%resolution%%save_dimension%%copy_dimension%%fps%%duration%%bitrate%%video_codec%%audio%%load_time align=right%
```

Токены:

| Токен              | Показывает                                                                       |
|--------------------|----------------------------------------------------------------------------------|
| `%file_position%`  | Позиция текущего файла среди поддерживаемых файлов в папке (например, `3 / 12`)  |
| `%filename%`       | Имя текущего файла                                                               |
| `%frames%`         | Общее число извлечённых кадров                                                   |
| `%frame_position%` | Текущий кадр / всего в режиме Single; в других режимах — только всего            |
| `%resolution%`     | Разрешение исходного видео (например, `1920x1080`)                               |
| `%fps%`            | Частота кадров источника                                                         |
| `%duration%`       | Длительность источника (`H:MM:SS`)                                               |
| `%bitrate%`        | Битрейт контейнера источника                                                     |
| `%video_codec%`    | Видеокодек источника                                                             |
| `%audio%`          | Сводка первого аудиопотока или `No audio`                                        |
| `%load_time%`      | Время, затраченное на извлечение отображаемых кадров (после извлечения)          |
| `%save_dimension%` | Прогнозируемые размеры вывода Save view                                          |
| `%copy_dimension%` | Прогнозируемые размеры вывода Copy view                                          |
| `%view_mode%`      | Активный режим просмотра (`Smart Grid`, `Grid`, `Scroll`, `Filmstrip`, `Single`) |
| `%zoom%`           | Активный режим масштаба                                                          |

Необязательные атрибуты внутри токена (в стиле xml, без пробелов в значениях):

| Атрибут | Значения                                 | Эффект                                                                                                                                       |
|---------|------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------|
| `width` | `auto` (по умолчанию) или пиксели        | Задаёт фиксированную ширину в пикселях. `auto` измеряет по тексту                                                                            |
| `align` | `left` (по умолчанию), `right`, `center` | Выравнивание текста панели                                                                                                                   |
| `cap`   | `true` (по умолчанию), `false`           | Только для `save_dimension` / `copy_dimension`: показывает форму после ограничения `WxH -> CxC`, когда CombinedMaxSide уменьшает изображение |

Токен, идентификатор которого записан полностью в верхнем регистре (например, `%VIEW_MODE%`), переводит свой отображаемый текст в верхний регистр. Идентификаторы со смешанным регистром (например, `%Resolution%`) оставляют текст как есть.

Когда данные токена недоступны:

- Панели с `width=auto` убираются из строки (соответствует прежнему поведению «нет панели для отсутствующих данных»).
- Панели с фиксированной `width=N` остаются на своём месте и показывают `?`, чтобы разметка не смещалась неожиданно.

Панели `%save_dimension%` и `%copy_dimension%` интерактивны: щелчок переключает соответствующую настройку *Save at view resolution* / *Copy at view resolution* (ту же, что отслеживает радио-переключатель в выпадающем списке). Настройка *Toolbar action* на вкладке Appearance определяет, срабатывает ли переключение по **одиночному** или **двойному** щелчку (по умолчанию — двойной щелчок). Курсор в виде руки при наведении и всплывающая подсказка панели указывают на эту возможность. **Ctrl+Click** по любой панели копирует её текст в буфер обмена независимо от этой настройки.

Настройка **Stretch auto-width panels to fill the bar** (вкладка Appearance, по умолчанию выключена) заставляет отрисовщик распределять оставшуюся ширину строки между панелями с автошириной пропорционально их естественной ширине — полезно, когда нужна выровненная по ширине строка без хвостового зазора. Панели с фиксированной `width=N` сохраняют свою ширину.

Индикатор выполнения по-прежнему существует вне шаблона — его положение относительно панелей задаётся настройкой *Progress bar position*.

Кнопка **Save view** на панели инструментов — это раздельная кнопка: щелчок по ней учитывает настройку *Save at view resolution*, а стрелка выпадающего списка открывает разовые пункты **Save view at view resolution...** и **Save view at native size...**, которые переопределяют настройку для этого единственного действия без сохранения. В старых версиях Windows (XP / Server 2003) выпадающий список вызывается правым щелчком по той же кнопке (системная стрелка выпадающего списка там недоступна); правый щелчок также работает на Vista+ как альтернативный способ доступа.

Кнопка **Copy view** на панели инструментов работает так же: раздельная кнопка, выпадающий список которой открывает разовые пункты **Copy view at view resolution** и **Copy view at native size**. Вариант с исходным размером повторно извлекает кадры в исходном разрешении перед публикацией в буфер обмена, так что буфер получает настоящие пиксели исходного разрешения, а не живые ячейки, масштабированные под область просмотра. Обычный щелчок Copy view учитывает собственную сохранённую настройку *Copy at view resolution* (отдельный переключатель от настройки Save view).

#### Hotkeys

Каждое командное действие в плагине настраивается. Каждому действию можно назначить любое число сочетаний — например, «Previous frame» по умолчанию привязано и к `Left`, и к `Ctrl+Left`, и вы можете добавить ещё (или удалить любое).

- Выберите строку в списке и нажмите **Assign…** (или дважды щёлкните по строке), чтобы открыть редактор сочетаний. Нажмите комбинацию клавиш, чтобы добавить её; выберите сочетание и нажмите **Remove**, чтобы удалить. Цифры Numpad и псевдонимы `+`/`-`/`.` сводятся к своим аналогам в верхнем ряду, так что привязка `0` автоматически охватывает `Numpad 0`.
- **Clear** стирает все сочетания, назначенные выбранному действию.
- **Reset all** восстанавливает привязки по умолчанию для всех действий.
- Конфликты разрешаются в момент назначения: когда вы добавляете сочетание, уже принадлежащее другому действию, редактор спрашивает, переназначить ли его. Согласие незаметно снимает сочетание с прежнего владельца.
- Tab, Alt+F4 и одиночные клавиши-модификаторы не могут быть переопределены.

Когда плагин удерживает фокус клавиатуры, все остальные комбинации клавиш принадлежат этой таблице, независимо от встроенных значений Lister по умолчанию. Непривязанное действие просто ничего не делает; исходные сочетания Lister (Escape для закрытия, `1`..`9` для переключения текстового/двоичного/шестнадцатеричного режимов и т. д.) по-прежнему доступны через меню Lister.

#### Диагностика

Для отладочного журналирования нет элемента в интерфейсе — переключатель редактируется вручную в файле `Glimpse.ini`:

```ini
[debug]
LogEnabled=1
```

Когда включено, плагин пишет в `glimpse_debug.log` рядом с WLX DLL. Файл очищается при каждом запуске Total Commander (когда журналирование включено), так что одна сессия воспроизведения умещается в один самодостаточный файл. Настройка вступает в силу при следующем запуске TC. Верните `0` (или удалите строку), чтобы остановить журналирование.

## Плагин WCX (Packer)

Представляет видеофайл как виртуальный архив, содержащий изображения-кадры. Открытие видео в TC показывает файлы вида `video_frame_001_00m05s.png`, которые можно копировать, просматривать или пакетно извлекать стандартными операциями TC.

### Сценарии использования

- Пакетное извлечение миниатюр сразу из многих видео с помощью многофайлового копирования TC
- Использование встроенного просмотрщика TC для просмотра отдельных кадров
- Запуск пользовательских пресетов ffmpeg (извлечение аудио, превью с низким битрейтом, произвольные перекодирования) путём их извлечения из виртуального архива — см. [Presets](#presets)

#### Presets

Пользовательские рецепты ffmpeg, которые появляются как дополнительные виртуальные файлы в списке архива наряду с записями кадров и объединённого изображения (или вместо них).

Включите отображение пресетов, отметив флажок **Presets** на вкладке Output. Отключённые пресеты остаются видимыми в редакторе (чтобы вы могли снова их включить), но не появляются в списке архива.

| Поле        | Описание                                                                                                                                                                                                                                                                                                                                                                    |
|-------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Name        | Имя секции INI; одновременно служит шаблонной переменной `%name%`. Должно быть уникальным (без учёта регистра)                                                                                                                                                                                                                                                              |
| Enabled     | Когда выключено, пресет скрыт из списка архива, но сохраняется в редакторе                                                                                                                                                                                                                                                                                                  |
| Description | Произвольная текстовая метка, отображается только в редакторе                                                                                                                                                                                                                                                                                                               |
| Output ext  | Расширение файла без ведущей точки (например, `mp3`, `jpg`). Обязательно. ffmpeg выбирает мультиплексор/кодировщик по нему, когда явный кодек не задан                                                                                                                                                                                                                      |
| Output name | Шаблон имени файла. По умолчанию `%basename%_%name%`. Распознаваемые переменные: `%basename%` (имя исходного файла без пути и расширения), `%name%` (имя секции пресета), `%ext%` (расширение источника в нижнем регистре, без точки). Неизвестные шаблоны `%token%` передаются как есть. Может содержать `/` или `\` для виртуальных подпапок внутри архива                |
| ffmpeg args | Аргументы, вставляемые между `-i <input>` и выходным файлом. Те же шаблонные переменные, что и в Output name, раскрываются и здесь (например, `-metadata title=%basename%`). Запрещённые токены (отклоняются при сохранении): `-i`, `-y`, `-n`, `pipe:0`, `pipe:1`, `pipe:2`. Может быть пустым — в этом случае ffmpeg выбирает кодеки по умолчанию из выходного расширения |

Пресеты хранятся в `presets.ini` рядом с `Glimpse.ini`. Файл можно редактировать вручную — каждый пресет представляет собой секцию `[name]` с `OutputExt=`, `Args=` и т. д. Редактор читает и записывает этот файл; ручные правки подхватываются при следующем открытии архива. Файл в UTF-8 (с BOM или без); современные редакторы сохраняют его корректно.

Когда извлечение пресета завершается ошибкой, фактическая ошибка ffmpeg отображается в диалоге («Glimpse preset extraction failed») с соответствующей строкой stderr ffmpeg.

Коллизии имён между пресетами получают стандартный суффикс `(N)` в списке; первый определённый получает имя без суффикса. Разместите пресет в виртуальной подпапке (например, `OutputName=audio/%basename%`), чтобы избежать коллизий и визуально сгруппировать связанные пресеты.

#### Size limit

Ограничивает большую сторону извлекаемого вывода в пикселях (ограничение применяется к той стороне, которая длиннее, независимо от ориентации). Полезно, чтобы пакетно извлекаемые миниатюры оставались компактными.

| Параметр                    | По умолчанию        | Описание                                                                                                                                 |
|-----------------------------|---------------------|------------------------------------------------------------------------------------------------------------------------------------------|
| Separate frames longer side | 0 (без ограничения) | Когда активен режим `Separate frames`, ffmpeg уменьшает каждый кадр так, чтобы его большая сторона не превышала указанное число пикселей |
| Combined image longer side  | 0 (без ограничения) | Когда активен режим `Combined image`, собранная сетка уменьшается так, чтобы её большая сторона не превышала указанное число пикселей    |

#### Диагностика

Для отладочного журналирования нет элемента в интерфейсе — переключатель редактируется вручную в `Glimpse.ini`:

```ini
[debug]
LogEnabled=1
```

Когда включено, плагин пишет в `Glimpse.log` рядом с WCX DLL. Строки помечаются `[WCX]` для общих операций и `[WCX-Presets]` для предупреждений загрузчика пресетов (отклонённые пресеты, некорректные значения). Настройка вступает в силу при следующем открытии архива — перезапуск TC не требуется. Верните `0` (или удалите строку) и снова откройте любой архив, чтобы остановить журналирование. Существующий файл журнала остаётся на месте для изучения.

## Установка

### Автоматическая (рекомендуется)

Откройте `.zip`-архив Glimpse в Total Commander. TC обнаружит `pluginst.inf` и предложит установить плагин автоматически. Устанавливайте архивы WLX и WCX по отдельности.

### Вручную

**WLX (Lister):**
1. Распакуйте `Glimpse.wlx` и `Glimpse.wlx64` в каталог по вашему выбору
2. В Total Commander: Configuration > Options > Plugins > Lister (WLX) > Add
3. Укажите `Glimpse.wlx64` (64-битный TC) или `Glimpse.wlx` (32-битный TC)

**WCX (Packer):**
1. Распакуйте `Glimpse.wcx` и `Glimpse.wcx64` в каталог по вашему выбору
2. В Total Commander: Configuration > Options > Plugins > Packer (WCX) > Add
3. Укажите `Glimpse.wcx64` (64-битный TC) или `Glimpse.wcx` (32-битный TC)
4. Свяжите нужные расширения видео (например, `mp4`, `mkv`)

## Требования

- Total Commander 10.x или новее
- `ffmpeg.exe` в каталоге плагина, по настроенному пути или в системном PATH

## Поддерживаемые форматы

MP4, MKV, AVI, MOV, WMV, WEBM, FLV, TS, M2TS, M4V, 3GP, OGV, MPG, MPEG, VOB, ASF, RM, RMVB, F4V — в основном всё, что поддерживает `ffmpeg`.

Дополнительные расширения можно настроить в настройках (F2 для WLX или через INI-файл для WCX).
